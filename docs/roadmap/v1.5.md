# v1.5 — Systeme de Plugins & CLI

**Date** : Mars 2026

---

## Apercu

La v1.5 transforme le projet d'une collection de skills en un **ecosysteme de plugins gerable**. Un CLI `mugiwara` permet d'installer, mettre a jour et gerer les agents individuellement. Chaque agent possede un manifest `mugiwara.yaml` avec metadonnees, checksums SHA256 et dependances. Un registre central (`registry.yaml`) indexe les 40 agents/pipelines.

## Taches detaillees

### 1. CLI Plugin Manager (`bin/mugiwara`)

**Agent** : `/usopp` (DevOps)

**Description** : CLI bash modulaire qui permet de gerer les agents comme des plugins :
- `mugiwara list` : lister les agents installes et disponibles
- `mugiwara install <agent>` : installer un agent depuis le registre
- `mugiwara uninstall <agent>` : supprimer un agent
- `mugiwara update` : mettre a jour tous les agents
- `mugiwara search <query>` : rechercher un agent par nom/description
- `mugiwara info <agent>` : afficher les details d'un agent

**Fichiers crees** :

| Fichier | Role |
|---------|------|
| `bin/mugiwara` | Point d'entree CLI |
| `lib/core.sh` | Fonctions utilitaires (couleurs, logging, erreurs) |
| `lib/registry.sh` | Gestion du registre (fetch, parse, cache) |
| `lib/manifest.sh` | Lecture/validation des manifests YAML |
| `lib/installer.sh` | Logique d'installation/desinstallation |
| `lib/cmd_install.sh` | Commande `install` |
| `lib/cmd_uninstall.sh` | Commande `uninstall` |
| `lib/cmd_update.sh` | Commande `update` |
| `lib/cmd_list.sh` | Commande `list` |
| `lib/cmd_search.sh` | Commande `search` |
| `lib/cmd_info.sh` | Commande `info` |

### 2. Manifests agents (`mugiwara.yaml`)

**Agent** : `/vegapunk` (Meta-Auditor)

**Description** : Chaque agent recoit un fichier `mugiwara.yaml` contenant :
- `name` : identifiant unique de l'agent
- `version` : version SemVer (1.5.0)
- `description` : description courte
- `category` : categorie (analysis, pipeline, security, data, etc.)
- `files` : liste des fichiers de l'agent
- `depends` : agents requis (pour les pipelines)
- `checksum` : hash SHA256 de chaque fichier

**Fichiers crees** : 40 fichiers `skills/*/mugiwara.yaml`

### 3. Registre central (`registry.yaml`)

**Agent** : `/vegapunk` (Meta-Auditor)

**Description** : Fichier index centralisé a la racine du projet. Permet au CLI de connaitre tous les agents disponibles sans scanner le filesystem. Contient pour chaque agent : version, description, categorie.

**Fichier cree** : `registry.yaml`

**Description** : Pre-calcul des checksums SHA256 de chaque SKILL.md dans les 40 manifests. Permet de verifier l'integrite des agents lors de l'installation et de detecter les modifications non autorisees.

## Risques identifies

| # | Risque | Impact | Probabilite | Mitigation |
|---|--------|--------|-------------|------------|
| 1 | CLI bash non portable (macOS vs Linux vs Windows) | CLI echoue sur certains OS | Moyenne | Tester sur les 3 OS dans CI, utiliser POSIX quand possible |
| 2 | Checksums invalides apres modification de SKILL.md | Installation echoue | Haute | Script de recalcul automatique des checksums |
| 3 | Registre desynchronise avec les agents reels | `mugiwara list` affiche des agents manquants | Moyenne | Hook pre-commit pour valider la coherence registre/agents |
| 4 | Dependances circulaires entre pipelines | Installation en boucle infinie | Faible | Detection de cycles dans `lib/installer.sh` |
