# v1.3 — Hooks Claude Code pour workflows post-agent

**Date** : 2026-03-02

---

## Apercu

Systeme complet de hooks Claude Code pour automatiser les workflows post-agent. Les hooks observent les invocations sans modifier les agents existants : logging JSONL, validation sync, notifications Slack optionnelles, et detection automatique de fin de pipeline.

---

## Nouveaux fichiers (8)

| Fichier | Role |
|---------|------|
| `.claude/settings.json` | Configuration centrale des hooks (events, matchers, async/sync) |
| `.claude/hooks/log-agent-output.sh` | Log chaque invocation d'agent dans `logs/agents.jsonl` (async) |
| `.claude/hooks/log-session.sh` | Log debut/fin de session dans `logs/sessions.jsonl` (sync) |
| `.claude/hooks/notify-slack.sh` | Forward notifications vers Slack webhook (async, optionnel) |
| `.claude/hooks/notify-complete.sh` | Detecte fin de pipeline → log + Slack (async) |
| `.claude/hooks/validate-agent-output.sh` | Valide output agent (non-vide, longueur, structure) → feedback sync |
| `.claude/hooks/run-smoke-tests.sh` | Lance smoke tests quand un fichier `skills/` est modifie (async) |
| `docs/roadmap/v1.3.md` | Ce fichier |

---

## Architecture des hooks

```
PostToolUse (matcher: Skill)
  ├── log-agent-output.sh      [async]  → logs/agents.jsonl
  └── validate-agent-output.sh [sync]   → additionalContext si pb

PostToolUse (matcher: Write|Edit)
  └── run-smoke-tests.sh       [async]  → logs/agents.jsonl (si skills/ modifie)

Stop
  └── notify-complete.sh       [async]  → logs/agents.jsonl + Slack

SessionStart
  └── log-session.sh           [sync]   → logs/sessions.jsonl + context injection

SessionEnd
  └── log-session.sh           [sync]   → logs/sessions.jsonl

Notification
  └── notify-slack.sh          [async]  → Slack (si SLACK_WEBHOOK_URL set)
```

---

## Principes de conception

- **Observation pure** : les hooks ne modifient aucun agent existant ni SKILL.md
- **Async par defaut** : logging et notifications ne bloquent jamais l'execution
- **Sync pour validation** : seul `validate-agent-output.sh` est synchrone (injecte du feedback)
- **Slack optionnel** : no-op si `SLACK_WEBHOOK_URL` n'est pas defini
- **Dependance unique** : `jq` pour le parsing JSON stdin
- **Format JSONL** : 1 objet JSON par ligne dans `logs/`
