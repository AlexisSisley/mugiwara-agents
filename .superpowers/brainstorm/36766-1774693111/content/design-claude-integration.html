Section 5 — OSC Interceptor & Intégration Claude Code

Détection intelligente de Claude Code — sans modifier son protocole

🔍 Stratégie de détection — 3 niveaux
NIVEAU 1
Process Tree Detection
Pour chaque PTY, le daemon inspecte l'arbre de processus fils.
Cherche : claude, claude-code, node .../claude-code/cli.js
Polling léger toutes les 2s via sysinfo crate.
→ Résultat : is_claude_active: bool per pane
NIVEAU 2
OSC Sequence Interception
Le vte parser du daemon filtre les séquences OSC en temps réel.
Claude Code émet des OSC pour : titre, statut, notifications, progress.

Séquences interceptées :
OSC 0;{title} → window/tab title (process name, cwd)
OSC 7;{uri} → current working directory
OSC 9;{text} → notification (desktop notification trigger)
OSC 1337;... → iTerm2-style: images, badges, marks
OSC 633;... → VS Code shell integration (command start/end/output)

→ Résultat : cwd, title, notifications, command boundaries
NIVEAU 3
Output Pattern Analysis
Analyse heuristique du texte affiché dans le terminal.
Détecte les patterns Claude Code sans dépendre des OSC.

Patterns reconnus :
╭──────────────────╮ → Claude Code box border = session active
⏺ Thinking... → status = Thinking
⏺ Using tool: Read → status = ToolUse("Read")
✓ Created file.rs → event: FileCreated
✓ Updated file.rs → event: FileUpdated
✗ Error: ... → status = Error
claude › → prompt ready = Idle

→ Résultat : ClaudeStatus { Idle | Thinking | ToolUse(tool) | Error(msg) }
📡 Event Bus — du daemon vers le GUI
ClaudeEvent enum :

enum ClaudeEvent {
  SessionDetected { pane_id, pid },
  SessionEnded { pane_id },
  StatusChanged { pane_id, status: ClaudeStatus },
  Notification { pane_id, title, body, urgency },
  FileModified { pane_id, path, action: Created|Updated|Deleted },
  ToolUsed { pane_id, tool_name, duration_ms },
  CommandBoundary { pane_id, command, exit_code, duration_ms },
  CwdChanged { pane_id, new_cwd },
}
Transport :
Les events sont envoyés via le control socket (même canal que les commandes).
Protocol : MessagePack avec tag type pour distinguer Command / Response / Event.
Le GUI s'abonne aux events au connect (Subscribe { event_types: Vec }).

Daemon ──[Event stream]──▶ GUI (async channel, non-blocking)
CLI peut aussi s'abonner : cmux events --follow --pane 3
