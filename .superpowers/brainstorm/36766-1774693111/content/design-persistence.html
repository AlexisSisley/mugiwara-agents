Auto-start :
Le GUI lance automatiquement le daemon s'il n'est pas déjà actif.
Détection via lock file : ~/.cmux/daemon.lock (contient PID)
Si le PID est mort → cleanup + relance. Si vivant → attach.
💾 Couches de persistance
Couche Quoi Où Quand
L1 — Live PTY processes vivants, shared memory RAM (daemon process) Tant que le daemon tourne
L2 — Session Layout complet (workspaces, tabs, splits, pane metadata) ~/.cmux/sessions/current.json Toute mutation (debounced 500ms)
L3 — Scrollback Historique terminal par pane (N lignes) ~/.cmux/scrollback/{pane_id}.bin Périodique (toutes les 30s) + on shutdown
L4 — Snapshot Screen buffer complet (ce qui est visible) ~/.cmux/snapshots/{pane_id}.bin On hibernate + on graceful shutdown

🛡️ Scénarios de recovery
Scénario Ce qui survit Recovery
GUI crash / close ✅ Tout — PTY vivants, scrollback, layout cmux attach → reconnexion instantanée
Daemon graceful shutdown ✅ Layout + scrollback + screen snapshots Auto-relance → restore layout, show snapshots, re-spawn PTY, re-exec last commands
Daemon crash (SIGKILL) ⚠️ Layout + scrollback (dernière save) Détection lock stale → cleanup → restore from last session JSON + scrollback files
Machine reboot ⚠️ Layout + scrollback + snapshots Daemon auto-start (registre Windows / systemd) → full restore avec proposition de re-exec commands
