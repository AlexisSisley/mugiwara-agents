Section 3 — Layout Engine & Workspaces

Workspaces → Tabs → SplitTree — 3 niveaux d'organisation

📐 Modèle de données — Hiérarchie
SessionState {
  workspaces: Vec<Workspace>,
  active_workspace: WorkspaceId,
}

Workspace {
  id: WorkspaceId,
  name: String, // "mugiwara-agents", "cmux-dev", "perso"
  cwd: PathBuf, // répertoire racine du workspace
  tabs: Vec<Tab>,
  active_tab: TabId,
  metadata: WorkspaceMeta, // git branch, project type, etc.
}

Tab {
  id: TabId,
  name: String, // auto-generated ou custom
  root: SplitNode, // arbre de splits
  focused_pane: PaneId,
}

SplitNode {
  Leaf { pane: PaneId },
  Split {
    axis: Axis, // Horizontal | Vertical
    ratio: f32, // 0.0..1.0 — position du séparateur
    left: Box<SplitNode>,
    right: Box<SplitNode>,
  }
}

Pane {
  id: PaneId,
  pty_id: PtyId, // lien vers le PTY dans le daemon
  cwd: PathBuf,
  title: String, // set by OSC title sequence
  claude_status: Option<ClaudeStatus>, // Idle | Thinking | ToolUse | Error
  notifications: Vec<Notification>,
  last_command: Option<String>,
}

⌨️ Opérations de layout
Action Keybind SplitTree mutation
New workspace Ctrl+Shift+N push Workspace { tabs: [default_tab] }
Switch workspace Ctrl+1-9 active_workspace = id
New tab Ctrl+T push Tab { root: Leaf(new_pane) }
Split right Ctrl+D replace Leaf(p) → Split { V, 0.5, Leaf(p), Leaf(new) }
Split down Ctrl+Shift+D replace Leaf(p) → Split { H, 0.5, Leaf(p), Leaf(new) }
Close pane Ctrl+W remove Leaf(p) → promote sibling
Focus direction Alt+←↑→↓ walk tree → nearest Leaf in direction
Resize split Alt+Shift+←↑→↓ parent.ratio ± 0.05
Command palette Ctrl+Shift+P fuzzy search all commands
